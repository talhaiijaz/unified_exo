What the simulator is doing

This page explains the physical model behind the CNT–Einzel workflow, the numerical choices inherited from the notebook/manuscript pipeline, and the deliberate simplifications used to keep the browser version interpretable and responsive.

1. Device model

The simulated system is an axisymmetric CNT forest macro-emitter followed by a three-electrode einzel lens. The emitter occupies a finite cylindrical region of radius Rforest and height hforest. The lens stack is defined by the aperture radius ainner, the outer stack radius rstack,out, the electrode thickness te, and the axial spacings d1, d2, d3.

The outer electrodes are grounded, the middle electrode is biased to VMID, and the cathode/forest conductor is held at -VCNT. An explicit annular oxide region can be included with relative permittivity εr.

2. Electrostatics

The static field solve is based on the axisymmetric variable-permittivity equation

\[ \nabla \cdot \left(\varepsilon(r,z)\nabla V(r,z)\right) = 0. \]

In cylindrical coordinates, the radial flux term carries the usual 1/r factor. The browser code keeps the notebook’s conservative finite-difference structure, using harmonic averaging of ε across cell faces so dielectric jumps are treated through interface fluxes rather than naive nodal averaging.

This matters because the oxide support is not a tiny correction. It reshapes the field near the aperture wall and therefore changes both extraction and focusing.

3. Field reconstruction for tracing

After solving for V(r,z), the browser page reconstructs the potential with a spline and takes derivatives from that spline when tracing particles. This is preferable to independently interpolating precomputed field arrays because it keeps the force law tied to a single scalar potential.

The signed transverse coordinate x used in the plots is just a meridional representation of the axisymmetric device:

\[ E_x(x,z)=\operatorname{sgn}(x)E_r(|x|,z), \qquad E_z(x,z)=E_z(|x|,z). \]

4. Relativistic ray equations

Even when launch energies are sub-eV, electrons are accelerated through kilovolt-to-tens-of-kilovolt structures. For that reason the browser model propagates momentum rather than using a purely nonrelativistic velocity update:

\[ \dot{x}=\frac{p_x}{\gamma m_e}, \qquad \dot{z}=\frac{p_z}{\gamma m_e}, \qquad \dot{p}_x=qE_x, \qquad \dot{p}_z=qE_z, \]
\[ \gamma = \sqrt{1 + \frac{p_x^2+p_z^2}{m_e^2 c^2}}. \]

The trajectories are integrated with DOP853 and terminate when a ray hits a conductor, exits radially, exits downstream, or turns around.

5. Launch model

The canopy is sampled face-by-face. Each launch face receives a Fowler–Nordheim / Murphy–Good-inspired current weight based on the local macro field just above the canopy. In this page, that weight is used in one of three modes:

  • Uniform radius for purely geometric sensitivity studies.
  • Emission quantiles to sample the weighted source with equal per-ray weights.
  • Emission all faces to retain the face weighting explicitly.

Angular and energy spreads can be delta-like or truncated Gaussian. The launch offset is applied slightly above the canopy to represent an effective post-extraction phase space rather than literal tunnelling at the surface.

6. Diagnostics

The main optical diagnostic is the weighted RMS beam radius

\[ \sigma_x(z)=\left(\frac{\sum_i w_i x_i(z)^2}{\sum_i w_i}\right)^{1/2}. \]

The page also reports:

  • count and weight transmission,
  • in-domain focus location,
  • asymptotic focus estimated from exit slopes,
  • fitted magnification,
  • maximum relative total-energy drift,
  • a lightweight feature-resolution table.

7. Design takeaways encoded into the page

The manuscript’s design studies motivate a few practical habits that are reflected in the interface:

  • Spot size should never be read without transmission.
  • Common voltage scaling is not a true similarity direction once source weighting is recomputed.
  • Moderate axial re-scaling can improve crossover formation more effectively than indiscriminate radial re-scaling.
  • The cathode-to-first-electrode spacing is a high-leverage parameter, so it deserves to be directly tunable.

The study page therefore surfaces transmission, focus, magnification, and energy drift together instead of reducing everything to one blur number.

8. What was simplified for the browser version

The notebooks include solver cross-checks, broader sweep workflows, cached publication figures, and more elaborate result bookkeeping. Those are intentionally not reproduced here. The browser build focuses on the core solve-and-trace loop:

  • single direct sparse solve instead of the full direct/BiCGSTAB comparison workflow,
  • no mesh or sweep-study automation,
  • parameter validation and feature-resolution checks instead of full convergence studies,
  • decimated plotting output so larger bundles stay visually responsive.

That keeps the site faithful to the physical workflow without turning the browser into a full publication notebook.
